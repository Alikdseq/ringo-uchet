## Шаблоны документов / политики

### 1. Соглашение об обработке персональных данных (RU GDPR-совместимое)

Разделы:
1. Общие положения (описание оператора, цели обработки: управление заявками, коммуникации, отчёты).
2. Перечень собираемых данных (ФИО, телефон, email, координаты, фото, документы).
3. Правовые основания (договор, согласие субъекта, закон 152-ФЗ).
4. Порядок и условия обработки:
   - Срок хранения, шифрование, доступ по ролям.
   - Передача третьим лицам (облачные провайдеры, почтовые сервисы) с указанием гарантий.
5. Права субъекта: запрос доступа, исправления, удаления, отзыв согласия.
6. Контакты ответственного за ПДн, порядок обращения.
7. Подпись/акцепт (чекбокс на сайте/в приложении + отдельный документ для оффлайн клиентов).

### 2. Cookie Policy / Политика конфиденциальности сайта

Разделы:
1. Какие данные собираются (cookie, аналитика, веб-формы).
2. Цели (улучшение сервиса, аналитика, персонализация).
3. Используемые сервисы (Google Analytics/Tag Manager, Yandex Metrica — если будут).
4. Управление cookie (как отключить, последствия).
5. Контакты и обновления политики.

### 3. Договор аренды/оказания услуг (скелет)

1. Стороны (Исполнитель — ООО «Ringo», Заказчик).
2. Предмет договора (аренда спецтехники, услуги операторов, доставка материалов).
3. Порядок согласования заявок (по системе, email, телефон).
4. Стоимость и порядок расчётов (предоплата, оплата по факту, штрафы за просрочку).
5. Сроки выполнения и ответственность сторон (SLA, форс-мажор).
6. Порядок приёмки (акты, фото, GPS подтверждение).
7. Конфиденциальность и обработка данных.
8. Заключительные положения (юрисдикция, изменение договора).

### 4. SLA документ

1. Условия реакции на заявки и инциденты (рабочее время, экстренная поддержка).
2. Времена ответа/решения (Response/Resolution Time) для типов запросов.
3. Каналы связи (почта, телефон, телеграм).
4. Метрики (доступность сервиса, latency).
5. Процесс эскалации.

### 5. DevOps/Support Runbook (outline)

1. Контакты ответственных (DevOps, backend, mobile, support).
2. Процедуры деплоя/роллбэка.
3. Бэкапы и восстановление.
4. Критичные алерты и действия.

Документы будут оформлены в `docs/contracts/` (Markdown + PDF). На данный момент зафиксирован состав и структура для быстрого согласования с юристами.
