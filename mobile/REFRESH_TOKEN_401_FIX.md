# Исправление ошибки 401 при обновлении токена

## Проблема

При истечении refresh token запрос на `/api/v1/token/refresh/` возвращал 401, который логировался в консоль браузера как ошибка, хотя это ожидаемое поведение и обрабатывалось автоматическим перелогином.

```
POST https://ringoouchet.ru/api/v1/token/refresh/ 401 (Unauthorized)
```

## Решение

Используется `validateStatus` в `auth_service.dart` для метода `refreshToken`, чтобы 401 не считался ошибкой Dio и не логировался в консоль как ошибка. Это ожидаемое поведение когда refresh token истек - приложение автоматически выполняет перелогин с сохраненными учетными данными.

### Изменения

**`mobile/lib/features/auth/services/auth_service.dart`**:
- Добавлен `validateStatus` в `Options` для запроса `/token/refresh/`
- 401 теперь обрабатывается как обычный ответ, а не как ошибка Dio
- 401 вручную проверяется и выбрасывается `UnauthorizedException`, который обрабатывается в `auth_notifier.dart`

## Результат

- 401 при истечении refresh token больше не логируется в консоль как ошибка
- Автоматический перелогин работает как и раньше
- Пользователь не видит ошибок в консоли - всё работает "тихо" в фоне

## Примечание

Браузер может все еще показывать 401 в Network tab (это нормально для отладки), но это не будет логироваться как ошибка в консоли через Dio/PrettyDioLogger.
